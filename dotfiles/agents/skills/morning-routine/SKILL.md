# Morning Routine

Runs the full morning setup for the vault: creates or updates today's daily note, fetches a fresh brief with weather/server/news, updates `PROJECTS.md`, and commits the vault to git.

This is an **orchestrator** — it calls the same logic as the `daily-note` and `daily-brief` skills rather than duplicating them.

## Trigger

Use this skill when:
- The user asks to "run morning routine", "start the day", "do the morning thing", or similar
- The user says `/skill:morning-routine`
- A session starts and nothing has been set up for today yet

## Requirements

- Same as `daily-note` and `daily-brief`: `curl`, `rg` (daily-brief only), SSH key to `100.75.139.39`
- `python3` for book cover frontmatter editing
- Vault root auto-detected from `$PWD` or `$VAULT_ROOT`
- `📌 Daily Brief.md` and `📰 Daily News.md` must exist at vault root
- Git must be initialized in the vault

## Step-by-step

### 1. Create or update today's daily note

Follow the same logic as the `daily-note` skill:

1. Resolve `TODAY`, `NOTE_PATH` as per daily-note step 1
2. Determine mode (create vs update) as per daily-note step 2
3. Fetch weather and server health as per daily-note step 3
4. If **create**: write the full note with tag link, date line, weather, Dataview task query blocks (Overdue, Due Today, School, Upcoming, No due date), empty sections (Tasks for today, What I did, Random), and live server check
5. If **update**: read existing note, preserve all user-written sections, only refresh weather and server check, bump frontmatter

Do **not** fetch news here — that's handled by the brief in step 2. Weather in the daily note is separate from weather in the brief — each has its own charming commentary.

### 2. Fetch the daily brief

Follow the same logic as the `daily-brief` skill:

1. Fetch weather (daily-brief step 2)
2. Fetch tasks from vault (daily-brief step 3) — independent call, do not share with step 1
3. Fetch server health (daily-brief step 4)
4. Fetch news headlines from BBC RSS (daily-brief step 5)
5. Write new day entry to `📌 Daily Brief.md` with weather, tasks, server, and #1-headline-per-category news (daily-brief step 6)
6. Write full 4-5 headlines per category to `📰 Daily News.md` (daily-brief step 6)
7. Set `unread: true` and bump `updated:` in frontmatter of both files

### 3. Update PROJECTS.md

**If the file doesn't exist**, create it:

```markdown
---
updated: {ISO_DATE}
unread: true
---

> [!info]- 🏗️ Project Dashboard
> Auto-generated by the morning routine. Create project notes in `600 Projects/` with `project:` frontmatter to see them here.

## Active

_No active projects yet._

## Completed

_None._
```

**If it exists**, regenerate the dashboard:

1. Scan `600 Projects/` for all `.md` files that contain `project:` in their frontmatter
2. Build a table from each project's frontmatter fields:
   - **Name** — wikilink to the project note
   - **Summary** — `summary:` field (first 60 chars)
   - **Status** — `status:` field
   - **Updated** — `updated:` field
3. If projects are found, replace the Active section with the table:

```markdown
| Project | Summary | Status | Updated |
|---------|---------|--------|---------|
| [[Server Stabilisation]] | Clean up unused services, lock down security | stalled | 2026-04-17 |
```

4. If no projects found (folder empty or no project frontmatter), leave the "No active projects" message
5. Move any completed projects (status: done/completed/archived) to the Completed section
6. Bump `updated:` in frontmatter, ensure `unread: true`

### 4. Download book covers

Scan all book notes in `200 Source Material/210 Books/` for `cover:` fields that still point to URLs (from the web clipper). Download each image to `_Attachments/` and swap the URL for a local wikilink.

```bash
BOOKS_DIR="$VAULT_ROOT/200 Source Material/210 Books"
ATTACH="$VAULT_ROOT/_Attachments"

for note in "$BOOKS_DIR"/*.md; do
  # Extract cover field value (handle both "url" and url)
  cover=$(grep -E '^cover:' "$note" | sed -E 's/^cover: "?([^"]+)"?/\1/')

  # Skip if not a URL
  echo "$cover" | grep -qE '^https?://' || continue

  # Build filename from note name
  slug=$(basename "$note" .md | tr '[:upper:]' '[:lower:]' | sed 's/ /-/g')
  local_file="cover-$slug.jpg"

  # Download
  curl -sL --max-time 15 -o "$ATTACH/$local_file" "$cover"

  # Only proceed if it's a real image
  if [ -s "$ATTACH/$local_file" ] && file "$ATTACH/$local_file" | grep -qE 'image|bitmap'; then
    python3 -c "
import re
path = '$note'
local = '$local_file'
iso = '$ISO_DATE'

with open(path) as f: content = f.read()

# Replace cover URL with wikilink
content = re.sub(r'^cover:.*', 'cover: \"[[{}]]\"'.format(local), content, flags=re.MULTILINE)

# Bump or add updated timestamp
if re.search(r'^updated:', content, re.MULTILINE):
  content = re.sub(r'^updated:.*', 'updated: ' + iso, content, flags=re.MULTILINE)
else:
  # Add after the opening ---
  content = content.replace('---', '---\nupdated: ' + iso, 1)

# Add unread if missing
if 'unread: true' not in content:
  content = content.replace('---', '---\nunread: true', 1)

with open(path, 'w') as f: f.write(content)
"
  else
    rm -f "$ATTACH/$local_file"
  fi
done
```

Skips notes that already have `[[wikilink]]` covers. Only processes URL-based ones. Safe to run daily — idempotent.

### 5. Commit to git

```bash
git -C "$VAULT_ROOT" add -A
git -C "$VAULT_ROOT" commit -m "morning routine - DD-MM-YY"
```

Do **not** push to remote (per vault convention). Let the commit naturally accumulate throughout the day; the next morning run will commit again.

## Example output summary

After running, the user has fresh versions of:
- **`500 Notes/520 Daily Notes/DD-MM-YY.md`** — their working document for the day
- **`📌 Daily Brief.md`** — compact overview with weather, tasks, server, news
- **`📰 Daily News.md`** — full headline list per category
- **`PROJECTS.md`** — project dashboard (created or refreshed)
- **Git history** — committed with morning-routine message
- **Book covers** — any URL-based covers in book notes get downloaded and swapped for local files

## Design notes

- Each step fetches its own data independently — no shared state between steps
- The individual skills (`daily-note`, `daily-brief`) remain fully functional on their own
- This skill is just wiring: sequence + PROJECTS.md + git
